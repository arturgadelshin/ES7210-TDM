# Сессия разработки ESP32-S3 Voice Assistant с TDM

## Дата: 17 мая 2026

## Цель
Заставить работать 4 микрофона (ES7210) через TDM режим ESP32-S3, при условии что на плате подключён только SDOUT1 от ES7210.

## Хронология

### Шаг 1: Исправление peak calculation (tdm_mic.cpp)
- **Проблема**: код использовал `bits_per_sample_` из YAML (32) вместо реальных 16 бит для расчёта stride в буфере
- **Исправление**: захардкодили 16-бит stride, исправили `frame_size` → `frame_sz` (опечатка), исправили `bits_per_sample_` → `16` в `start_driver_()`
- **Результат**: скомпилировано, прошито

### Шаг 2: Первый тест — MIC1 и MIC2 работают, MIC3/MIC4 = 0
Логи показали:
```
TDM 8x16 peak - S0: 85 S1: 32256 S2: 0 S3: 0 S4: 112 S5: 32768 S6: 0 S7: 0
```
S0 (MIC1) и S4 (MIC2) имеют сигнал, S2 (MIC3) и S6 (MIC4) = 0.

### Шаг 3: Исследование ES7210 TDM регистра 0x12
- Исследовали ESP-BSP драйвер от Espressif
- Подтвердили: reg 0x12 = 0x02 правильно для 4-канального TDM на SDOUT1
- **Ключевое открытие**: ESP-BSP драйвер НЕ пишет в CLOCK_OFF_REG01 (0x01), но наш драйвер писал 0x3f (выключал все часы)
- Кроме того, `configure_mic_gain_()` с маской 0x0b пропускал bit 2 (ADC3 clock)

### Шаг 4: Исправление ES7210 инициализации
Переписали `es7210_tdm.cpp` по последовательности ESP-BSP:
- Убрали запись `0x3f` в CLOCK_OFF_REG01
- Убрали запись в MODE_CONFIG_REG08
- Упростили `configure_mic_gain_()` — пишем `gain | 0x10` напрямую в регистры 0x43-0x46
- Добавили register dump для диагностики
- **Результат**: все 4 микрофона заработали!
```
TDM 8x16 peak - S0: 232 S1: 32000 S2: 14 S3: 32512 S4: 222 S5: 32512 S6: 263 S7: 32768
```

### Шаг 5: Проблема — DMA зависает после остановки спикера
После воспроизведения TTS ответа, данные микрофона становились статичными:
```
S0: 29 S1: 14592 S2: 6 S3: 14080 S4: 47 S5: 7680 S6: 38 S7: 8704 (повторяющийся)
```
Wake word переставал обнаруживаться.

### Шаг 6: Поиск корня проблемы — конфликт I2S портов
- Изучили исходники `i2s_tdm_audio` и `i2s_audio` компонентов
- Обнаружили: `i2s_tdm_audio.h` имел `port_{0}` (хардкод), `i2s_audio` тоже назначал порт 0
- **Оба I2S (mic и speaker) были на одной аппаратной периферии ESP32-S3!**
- При остановке спикера ломался DMA микрофона

### Шаг 7: Исправление — I2S порт 1 для TDM
- Изменили `i2s_tdm_audio.h`: `port_{0}` → `port_{1}`
- ESP32-S3 имеет 2 независимых I2S периферии
- **Результат**: микрофон продолжает работать после остановки спикера

### Шаг 8: Финальное тестирование
- Wake word обнаруживается надёжно (0.79-0.89 probability)
- STT распознаёт речь корректно ("погода" → "Сейчас плюс 23,7, частичная облачность")
- Voice assistant pipeline работает полностью
- Микрофон продолжает работать после нескольких циклов спикера

## Ключевые уроки
1. ESP-BSP драйвер от Espressif — лучший референс для ES7210, не ESPHome стандартный драйвер
2. ESP32-S3 имеет 2 I2S периферии — при использовании TDM + стандартного I2S нужно разные порты
3. ES7210 TDM рег. 0x12=0x02 мультиплексирует все 4 канала на SDOUT1 (без дополнительных настроек)
4. `CLOCK_OFF_REG01` = 0x3f выключает ВСЕ часы ADC — не пишите это при 4-канальном TDM
